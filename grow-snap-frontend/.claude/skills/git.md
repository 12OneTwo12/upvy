# Git 전략 가이드

GrowSnap 프로젝트의 Git 커밋 및 PR 규칙입니다.

## 커밋 메시지 형식

### 기본 구조

```
<type>(<scope>): <subject>

<body> (optional)

<footer> (optional)
```

### Type 목록

| 타입 | 설명 |
|------|------|
| **feat** | 기능 추가 |
| **fix** | 버그 수정 |
| **docs** | 문서 수정 |
| **style** | 코드 포맷 변경 (로직 변경 없음) |
| **refactor** | 리팩토링 |
| **test** | 테스트 코드 추가 또는 수정 |
| **chore** | 빌드 관련 수정, 의존성 관리 등 |

### 작성 예시

```
feat(auth): 사용자 인증 기능 추가

사용자 로그인 및 인증 기능을 추가했습니다.

Closes #123
```

### 커밋 규칙

- 제목은 50자 이내, 본문은 72자 이내
- 마침표 `.` 사용 금지
- 커밋이 많아질 경우 squash 활용

## Pull Request 템플릿

```markdown
### 작업 내용
[간단한 설명]

---

### 관련 이슈
Closes #이슈번호

---

### 작업 사항
- [변경 요약]
- [추가 요약]

---

### 리뷰 요청 사항
[리뷰어에게 요청할 사항]
```

## 브랜치 네이밍

| 브랜치 유형 | 형식 예시 |
|------------|----------|
| 기능 | `feature/ISSUE-13` |
| 배포 | `release/v1.2.0` |
| 긴급 수정 | `hotfix/ISSUE-14` |

## 이슈 연결

- `Closes #이슈번호`: PR 머지 시 이슈 자동 종료
- `Fixes #이슈번호`: 버그 수정 시 사용
- `Relates to #이슈번호`: 관련 이슈 (자동 종료 안됨)
